Creating a custom element|创造自定义元素

Web Components 最重要的是自定义元素。customElements API为我们提供了一个定义自定义HTML标记的路径，可以在包含定义类的任何文档中使用。

可以把它想象成React或Angular组件（例如 <MyCard />），但没有React或Angular依赖。 原生自定义元素如下所示：<my-card> </ my-card>。 更重要的是，将它视为一个标准元素，可以在你的React，Angular，Vue 等 应用程序中使用，省去不必要的麻烦。

本质上，自定义元素由两部分组成：标记名称和扩展内置HTMLElement类的类。 我们的自定义元素的最基本版本如下所示：

在上面的示例中，我们定义了一个新的符合标准的HTML元素，<one-dialog> </ one-dialog>。 它没有做太多…但是。 现在，在任何HTML文档中使用 <one-dialog> 标签将创建一个新元素，其中<h1>标签为“Hello，World！”

我们肯定会想要更强大的东西，我们很幸运。 在上一篇文章中，我们研究了为对话框创建模板，因为我们可以访问该模板，所以让我们在自定义元素中使用它。 我们在该示例中添加了一个脚本标记来执行一些对话魔术。 让我们暂时删除它，因为我们将把我们的逻辑从HTML模板移动到自定义元素类中。

现在，我们定义了自定义元素（<one-dialog>），并指示浏览器呈现HTML模板中包含调用自定义元素的内容。
