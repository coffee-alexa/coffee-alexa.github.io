什么是web组件？

web组件由三种独立的技术组成，这些技术可以一起使用：

  1. 自定义元素，很简单，这是完全有效的HTML元素，包括使用一组JavaScript API 制作的自定义模板，行为和标记名称（e.g.<one-dialog>）。defined in the HTML Living Standard specification.
  2. Shadow DOM，能隔离CSS和JavaScript，像是<iframe>。 defined in the Living Standard DOM specification
  3. HTML模板，HTML中用户自定义的模板，在调用之前不会呈现。defined in the HTML Living Standard specification.

这些是构成web组件的规范

Web组件通常在大多数主流浏览器都是可用的，除了Microsoft Edge and Internet Explorer 11，但是他们的存在填补了空白。

参考任何一个web组件都是技术全面的，因为他的术语有些过载了。因此，每种技术都可以独立使用或与其它技术组合。换句话说，他们并不排斥。

让我们快速浏览一些前三个小点。我们将在本系列的其他文章中深入探讨它们。

自定义元素 (Custom elements)

正如名字暗示的，自定义元素是HTML元素，例如<div> <section> 或 <article>，但我们可以通过浏览器API定义自己的名称。自定义元素很像那些标准HTML元素——写在尖括号里面——除了总是有一个破折号，比如 news-slider 或者 bacon-cheeseburger 。浏览器厂商已承诺不会创建名称中包含短划线的新内置元素，以避免冲突。

自定义元素包含自己的语义、表现、标记 并且可以跨框架和浏览器共享

自定义元素不依赖第三方框架而存在，并且浏览器供应商应致力于规范的持续向后兼容，除了保证根据规范编写的组件不会破坏API的更改。此外，这些组件通常可以和现有的最流行的框架一起使用，包括Angular，React，Vue和其他很少的一部分。

Shadow DOM

shadowDOM是DOM的封装版本。这允许开发者有效地将DOM片段彼此隔离，他们可以被css选择器选择，并添加相关联的样式。通常，document范围内的任何内容都被称为light DOM，而shadow root中的任何内容被称为shadow DOM

使用light DOM时，可以使用document.querySelector(‘selector’)选择某个元素或使用element.querySelector(‘selector’)选择某个元素的子元素。以同样的方式，可以通过shadowRoot.querySelector来定位shadow root 的子节点，来自对docuemt片段的参考——不同之处在于shadow root的子节点不能从light DOM中选择。例如，如果我们有一个带有shadow root的<button>，调用shadowRoot.querySelector（‘button’）将返回我们的按钮，但是不调用该文档的查询选择器将返回该元素，因为它属于不同的 DocumentOrShadowRoot实例。样式选择器的工作方式同理。

在这方面，shadowDOM的工作类似于<iframe>，其中内容与文档的其余部分隔离; 但是，当我们创建一个shadowRoot时，我们仍然可以完全控制页面的那一部分，但是作用于上下文。 这就是我们所说的封装。

如果您曾编写过重用相同内容的组件或依赖于CSS-in-JS工具或CSS命名策略（如BEM），那么shadow DOM有可能改善您的开发人员体验。
