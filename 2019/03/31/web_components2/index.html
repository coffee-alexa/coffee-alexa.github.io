制作可重复使用的HTML Template

在上一篇文章中，我们从高层讨论了Web组件规范（自定义DOM，shaodw DOM，HTML template）。在本文和接下来的三篇文章中，我们将对这些技术进行实验并更加i详细地研究他们。并了解我们如何在生产中使用它们。为此我们将从头开始构建自定义模式对话框，以了解各个技术是如何组合一起的。

HTML templates

Web Components规范中最不被认可但最强大的功能之一是<template>元素。在本系列的第一篇文章中，我们将模板元素定义为“HTML中用户定义的模板，直到被调用时才会呈现”。换句话说，模板时浏览器忽略的HTML，直到被告知不这样做。

然后，这些模板可以以很多有趣的方式传递和重用。出于本文的目的，我们将介绍为最终将在自定义元素中使用的对话框创建模板

真正的魔法发生在document.importNode方法中。 此函数将创建模板内容的副本，并准备将其插入到另一个文档（或文档片段）中。 函数的第一个参数抓取模板的内容，第二个参数告诉浏览器对元素的DOM子树（即其所有子节点）进行深层复制。

我们可以直接使用template.content，但这样做我们就会从元素中删除内容并稍后附加到文档的主体。 任何DOM节点只能在一个位置连接，因此后续使用模板的内容将导致空文档片段（基本上为空值），因为之前已迁移了内容。 使用document.importNode允许我们在多个位置重用相同模板内容的实例。
